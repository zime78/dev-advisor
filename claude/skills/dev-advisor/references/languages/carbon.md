# Carbon

원천: `docs/PROGRAMMING_LANGUAGES_2026.md`, 기준일 2026-05-13. 최신 순위와 시장 지표는 답변 시점에 웹으로 확인한다.

## 핵심 판단

Google이 주도하는 실험적 C++ 후계 언어로, C++ 완전 상호운용을 최우선으로 설계했다. 2026년 현재 프로덕션 사용은 이르다. Carbon 도입 판단은 문법 선호보다 실행 환경, 기존 자산, 운영 인력의 숙련도에 더 크게 좌우된다. 제품의 주 경로가 C++ 코드베이스의 점진적 현대화, 시스템 프로그래밍 실험, C++ 후계 언어 연구이고, 다음 제약을 통제할 수 있다면 우선 검토할 만하다: .

## 사용처

C++ 코드베이스의 점진적 현대화, 시스템 프로그래밍 실험, C++ 후계 언어 연구. 실무에서는 이 범위를 넘어 모든 문제를 해결하는 범용 선택지로 보기보다, Carbon 생태계가 이미 강한 플랫폼과 도구 체인에 맞춰 채택하는 편이 안정적이다. 신규 프로젝트에서는 장기 유지보수 인력과 배포 대상의 제약을 함께 확인해야 한다.

## 특징

C++ 양방향 interop, 명시적 소유권·안전성, 제네릭 시스템 재설계, 간결한 문법. 코드 작성 모델, 빌드 방식, 런타임 제약이 프로젝트 구조에 직접 영향을 주므로 초기 설계 단계에서 테스트 전략과 패키지 관리 방식을 같이 정해야 한다.

## 장점

. 특히 기존 생태계의 표준 도구를 그대로 활용할 수 있을 때 생산성과 운영 예측 가능성이 높다. 팀이 관용구와 디버깅 방식을 익히면 같은 기능을 더 적은 접착 코드로 구현할 수 있다.

## 제약

. 이 제약은 작은 실험에서는 드러나지 않다가 배포, 성능, 보안 감사, 장기 유지보수 단계에서 비용으로 나타나는 경우가 많다. 도입 전에는 대체 언어와 운영 모델을 같이 비교해야 한다.

## 적합한 프로젝트

- C++ 레거시 코드베이스를 장기적으로 현대화할 연구 또는 파일럿 프로젝트
- C++ interop이 필수인 환경에서 언어 선택지 평가
- Carbon 표준 도구와 런타임을 팀이 직접 운영할 수 있고, 코드 리뷰에서 언어별 관용구를 일관되게 적용할 수 있는 프로젝트.

## 부적합하거나 주의할 프로젝트

- 당장 출시가 필요한 모든 프로덕션 프로젝트
- C++와 무관한 새 프로젝트 — Rust, Zig, Go가 훨씬 성숙
- 소규모 팀 — 불안정한 언어 사양 추적 비용이 큼
- Carbon 경험자가 없는데 핵심 장애 대응, 성능 튜닝, 릴리스 자동화까지 동시에 요구되는 프로젝트.

## 학습 난이도와 선행 지식

C++ 숙련자라면 문법 변화보다 개념 변화(명시적 소유권, 제네릭 재설계)에 집중하면 된다. C++ 없이 Carbon을 먼저 배우는 것은 권장하지 않는다. 입문 단계에서는 문법보다 작은 프로그램을 빌드, 테스트, 포맷, 배포하는 전체 흐름을 먼저 익히는 것이 좋다. 실무 투입 전에는 오류 처리, 의존성 관리, 표준 라이브러리 사용법을 팀 규칙으로 정리해야 한다.

## 운영/배포 관점

LLVM 기반으로 네이티브 바이너리 생성. C++ 빌드 시스템(Bazel, CMake)과 통합을 목표로 하지만 현재 지원 수준은 실험적이다. 재현 가능한 빌드, 의존성 잠금, 런타임 버전 고정, 관측 로그 포맷을 초기에 정하지 않으면 운영 환경 차이로 인한 결함이 늘어난다.

## 타입/런타임 특성

정적 타입, AOT 컴파일, GC 없음. C++와 동일한 메모리 모델. 안전성은 컴파일 타임 검사와 런타임 체크 혼용 예정. 이 특성은 API 경계, 테스트 범위, 병렬성 모델, 장애 격리 방식에 영향을 준다. 언어의 타입 시스템이 보장하지 않는 부분은 정적 분석, 테스트, 코드 리뷰 규칙으로 보완해야 한다.

## 실사용 예제

다음 예제는 이 언어를 단순 출력이 아니라 실제 업무 코드에서 자주 나오는 데이터 집계, 자동화, 런타임 제어, 또는 플랫폼 특화 작업에 적용하는 형태로 보여준다.

```carbon
package RevenueReport api;

class Order {
  var region: String;
  var amount: i32;
  var paid: bool;
}

fn PaidRevenue(orders: Slice(Order)) -> i32 {
  var total: i32 = 0;
  for (order: Order in orders) {
    if (order.paid) {
      total += order.amount;
    }
  }
  return total;
}
```
