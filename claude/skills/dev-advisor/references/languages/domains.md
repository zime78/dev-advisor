## 선택 기준

- 문제 영역: 제품이 주로 다루는 런타임, 데이터, 플랫폼, 성능 요구를 먼저 본다.
- 팀 역량: 현재 팀이 운영 가능한 언어와 신규 학습 비용을 함께 본다.
- 배포 환경: 클라우드, 브라우저, 모바일, 임베디드, 메인프레임, DB 내부 실행처럼 제약이 강한 환경을 확인한다.
- 생태계: 프레임워크, 패키지, 테스트, 관측, 배포, 보안 도구의 성숙도를 본다.
- 유지보수 기간: 장기 운영 시스템일수록 인력풀과 호환성을 더 크게 본다.
- 공식 근거: 최종 도입 전에는 상세 파일의 `## 관련 문서`에서 공식 문서, 언어 레퍼런스, 패키지/툴링 문서를 확인한다.
